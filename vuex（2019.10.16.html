Vuex 能处理任何情况下的组件信息通信

前提：SPA单页面 （实现同一个页面中，各组件的信息通信）

vuex：vue中实现公共状态管理的插件 ，每一个插件都是一个类。

通过全局方法 Vue.use（vuex）使用插件  基于mixin混入的方式给vue提供一个store



new Vue.Store({})   //创建store容器（创建Vue下Store这个类的实例

以下是一个表示“单向数据流”理念的简单示意：（视图里通过点击，触发actions行为操作，actions行为操作改变State状态，State状态改变，视图改变）

Vuex原理

实现 组件A 和 组件B 之间的信息通信：

1、new Vuex.Store（{....}）：Vuex插件首先会帮我们创建一个Store（公共状态管理池），存放组件之间需要共享的公共状态（state(DATA)：n: 10 m: 20）。

2、获取：组件A 可通过 this.$store.state.n/m 获取容器中的状态信息，把状态信息绑定在视图上。

3、修改操作：同步修改操作 和 异步修改操作

4、mutation：存放所有同步修改store中状态信息的action行为（方法），handle（state）{同步修改容器中的状态信息 }

5、组件A 通过this.$store.commit（'handleN') 让 mutation 中的方法执行，从而同步改变容器中的状态信息（管理员模式）

6、需求：在组件B 中，我们需要点击某个按钮，从服务器先异步获取数据，把获取的数据修改公共状态m

7、actions：存放所有异步修改state的方法

8、能够修改store中的公共状态只有mutation这个入口

9、在组件B 中点击某个按钮，触发 actions 中的queryData（store）{ 先异步获取数据store.commit（’handleM’） }

10、组件B 调取 actions 中的方法（this.$store.dispatch（’queryData’）执行actios中的queryData方法)

 store  英[stɔː(r)]：商店、贮存物、存储



 dispatch  英[dɪˈspætʃ]: 派发、派遣、发出

关于子组件不重新渲染最新数据的问题

在beforeCreate-created之间：创建一个实例，把 data挂载到实例上，并进行处理，触发created生命周期函数，组件如果不销毁，data只执行一次，父组件更新，触发子组件的 updated，子组件渲染的时候 data 不会再执行（只有第一次渲染子组件，在beforeCreate 和 created 中间的时候执行一次）

解决方法

在子组件中使用methods、computed、watch方案，只要父组件重新渲染子组件，就会把最新的属性传递进来。

在父组件调用的子组件中设置v-if

<template>

  <div>

    <temp2 :n="n" :m="m" v-if="flag"></temp2>

  </div>

</template>



<script>

import temp2 from "./temp2";

export default {

  components: {

    temp2

  },

  data() {

    return {

      n: 0,

      m: 0,

      flag:true

    };

  },

  mounted() {

    setTimeout(() => {

        this.flag = false;

        //父 beforeUpdate -> 子 beforeUpdated -> 子updated -> 父 updated

      this.n = 100;

      this.m = 200;

      this.$nextTick(()=>{

          this.flag = true 

      })

    }, 1000);

  }

};

</script>

function mapMutation(arr){

   let obj = {}

arr.forEach(item=>{

  obj[item] = function(...args){

        this.$store.commit(item, ...args);

       }

Vuex按照模块进行状态管控和标准化写法

创建模块 person.js 、 product.js ，在 index.js 中把每一个模块中的state~actions都进行合并

state 会按照各板块进行区分 state={Person:{…},product:{…},isLogin:true}

但是 getters / mutations / actions 默认不会进行模块区分，默认是全部合并在一起，我们想要按照 state 这种方式来区分，解决方案：每个设置 namespaced: true；这样最后虽然也是把每个模块中的方法合并在一起了，但是会以模块的名字作为前缀，来进行标识和区分 mutations={‘person/changeName’:function xxx, product/changeName’:function xxx,changeLogin/changeName’:function xxx,}

Vuex 实现数据存储

浏览器刷新，页面重新渲染，此时我们需要

  • 从服务器基于ajax请求获取数据（所有的任务信息）；

  • 把获取的信息存储到vuex管控的store容器中

  • 组件中需要的数据都从store.state中获取使用（做个判断：如果没有数据，我们派发一个任务，从服务器重新获取数据，再放到vuex中即可）

问题：新增任务后

  • 把新增的任务给服务器存储（服务器数据更新）

  • 此时我们一定得需要让vuex中的数据和服务器的数据保持统一（1、重新派发一个请求，从服务器获取最新的数据，同步给vuex中的数据（推荐）；2、也可以不操作服务器，我们在原始vuex的数据中进行处理也可以）；修改删除也是如此。

把数据绑定在页面上，点击选项卡切换按钮，都会重新从服务器拉取对应状态的任务信息

未完成  ->  ajax

已完成  ->  ajax

全部    ->  ajax

未完成  ->  ajax



频繁切换会增加服务器的压力，我们想：页面刷新，我们从服务器获取数据，把数据临时存储在一个地方AAA；页面从AAA里面获取数据，展示在容器中，并且当我们频繁切换的时候，也都是从AAA中获取数据重新筛选，然后呈现（当前这里需要做一个事情，如果AAA中没有数据，我们重新发ajax请求，从服务器获取最新的数据存储在AAA中即可）



当用户手动刷新的时候，我们应该还是重新从服务器获取最新的数据





AAA   -> 本地存储localStorage

1、存储的信息都是字符串，每次还需要不断解析处理

2、永久性存储，页面刷新也不会消失
